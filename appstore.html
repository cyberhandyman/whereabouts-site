Apple Developer 注册 + 双平台上架手册

给 Bam 的一站式操作手册:从付 ¥688 年费到「何处」同时出现在 iOS 和 Mac App Store。工程侧的准备(双 target、合规 key、entitlements、图标、隐私政策网页)已经全部做好,你只需要走完下面这些必须本人操作的步骤。事实核对于 2026-07。

① 注册开发者 ② Xcode 配置 ③ 工程已备好的 ④ 建 App record ⑤ 元数据文案 ⑥ 截图 ⑦ 提交审核 ⑧ 风险与对策 ⑨ 大陆区 & ICP

① 注册 Apple Developer(必须本人,约 1–3 天)

  1. 确认 Apple ID 已开双重认证

    设置 → 你的名字 → 登录与安全 → 双重认证。没开的先开,注册硬性要求。

  2. 用 iPhone 上的「Apple Developer」app 注册(推荐)

    App Store 搜「Apple Developer」→ 登录 Apple ID → Enroll / 注册。过程中会要求拍身份证 + 人脸验证,姓名务必用身份证上的法定姓名(拼音),用昵称会被卡审核。

    网页入口备选:developer.apple.com/programs/enroll。个人(Individual)即可 —— 卖家名会显示你的姓名;不需要 D-U-N-S(那是公司注册用的)。

  3. 付年费 ¥688/年($99)

    App 内注册直接用你 Apple 账户绑定的支付方式扣款 —— 大陆账户支持支付宝 / 微信支付 / 银联卡。注意不能用礼品卡余额。自动续费,可随时在订阅里取消。

  4. 等审核通过邮件

    个人注册一般 48 小时~5 个工作日。期间 Apple 可能打验证电话/发邮件,48 小时内要回复,否则会拖慢。通过后 App Store Connect 就能登录了。

    免费 app 不需要签 Paid Applications 协议、不需要填银行税务信息 —— 默认的会员协议就够了。

② 注册通过后的 Xcode 一次性配置(10 分钟)

  1. Xcode 登录开发者账号

    Xcode → Settings → Accounts → 「+」→ 登录你的 Apple ID。登录后能看到你的 Personal Team 变成付费 Team。

  2. 把 Team ID 写进 project.yml

    Accounts 页面点你的 Team → 右侧能看到 Team ID(10 位字母数字)。打开项目根目录 project.yml,把 DEVELOPMENT_TEAM: "" 改成 DEVELOPMENT_TEAM: "你的TeamID",然后跑一次 xcodegen generate。之后两个 target 都会自动签名。

  3. 给 macOS target 挂上 App Store 沙箱 entitlements

    工程里已备好 Whereabouts/Whereabouts-AppStore.entitlements(沙箱 + 网络 + 用户选择文件读写)。App Store 版必须开沙箱。在 project.yml 的 Whereabouts target settings 里加一行再重新生成:

    CODE_SIGN_ENTITLEMENTS: Whereabouts/Whereabouts-AppStore.entitlements
    ⚠️ 开沙箱后 app 的数据目录会变(搬进 ~/Library/Containers)。你自己 Mac 上的旧数据不会自动迁移 —— 上架前记得先在旧版里「设置 → 数据 → 导出 JSON」,装了沙箱版后再导入。新用户无感。
    ⚠️ 另一个上架前建议:目前 API key 明文存在偏好设置里(开发期为了 ad-hoc 签名不弹钥匙串框的取舍)。正式签名稳定后建议把 AI/Keychain.swift 换回真 Keychain 实现(Security 框架),几十行的事,不做也能过审(隐私政策里已如实说明)。

③ 工程侧已经准备好的东西(无需操作,给你个底)

  • 双 target:Whereabouts(macOS)+ WhereaboutsiOS(iOS),同一个 bundle id com.bamcope.whereabouts → 满足 universal purchase(一条 App record 挂两个平台,用户一次获取全平台)。
  • Info.plist 合规:LSApplicationCategoryType(效率工具)、ITSAppUsesNonExemptEncryption=false(只用系统 HTTPS → 出口合规豁免,提交时不用再答加密问卷)。
  • 图标:macOS .icns + iOS 1024 单尺寸 asset 都在。
  • 隐私政策网页(中英,App Store 必填 URL)已上线,见 ⑤ 的链接表。
  • iOS 截图辅助:DEBUG 版支持 --demo-data / --open-first / --tab-record / --tab-settings 启动参数,拍图不用手动造数据。
  • 版本号 0.2.0 / build 1,双端一致。

④ 在 App Store Connect 建 App record(15 分钟)

  1. 注册 bundle ID

    Certificates, Identifiers & Profiles → Identifiers → 「+」→ App IDs → App → Bundle ID 选 Explicit,填 com.bamcope.whereabouts。Capabilities 全部默认即可(本 app 不用推送/iCloud)。

  2. 新建 App(一条 record,两个平台)

    App Store Connect → My Apps → 「+」→ New App → 平台先勾 iOS,名称、主语言(简体中文)、bundle id、SKU(随便,如 whereabouts-001)。建好后在 app 页面左侧「Add Platform」再加 macOS —— 同一条 record,这就是 universal purchase 的正确姿势。

    app 名称「何处」两个字大概率被占用 —— 名称在每个地区唯一。备选:「何处 - 东西放哪了」「何处 Whereabouts」。名称可在提交前随时改。
  3. 价格与销售范围

    Pricing → 免费(0 元档)。Availability:建议首发去掉「中国大陆」,其余全选(原因见 ⑨)。

⑦ 打包上传 + 提交审核

  1. Archive 两个平台 
    cd 项目目录 && xcodegen generate
# Xcode 里: scheme 选 WhereaboutsiOS → destination "Any iOS Device" → Product ▸ Archive
# 再切 scheme Whereabouts → "Any Mac" → Product ▸ Archive

    Organizer 弹出后对每个 archive 点 Distribute App → App Store Connect → Upload,全部默认下一步。首次会自动帮你建发行证书。

  2. TestFlight 自测(强烈建议)

    上传后 10–30 分钟构建出现在 App Store Connect → TestFlight。自己手机装 TestFlight 跑一遍核心流程(录入 / 详情 / AI / 导入导出)。

  3. 填版本信息并提交

    每个平台各自的版本页:选构建、贴截图、贴描述/关键词、填审核备注(下节模板)→ Submit for Review。两个平台可以分别提交,先后无所谓;两个平台都过审后 universal purchase 自动生效。审核一般 1–3 天。

⑧ 审核风险与对策(重点看)

唯一值得认真对待的风险:BYOK(用户自带 API key)。有判例(Apple 开发者论坛 thread 763884)被以 Guideline 3.1.1 拒审,理由是「API key 是 IAP 之外解锁功能的机制」;但 App Store 上同类 BYOK app 大量在架,执行并不一致。「何处」的有利点:app 完全免费、不配 key 也是全功能(本地解析一切可用),AI 只是可选增强,没有任何「解锁/付费墙」语义。

审核备注(App Review Notes)直接粘贴:

This app is completely free with all features fully functional offline.
The optional "AI understanding" feature lets users bring their own API key
(Anthropic or Volcengine) purchased directly from those providers for their
own use. The key unlocks nothing in this app — it only lets the app send the
user's own item text to the user's own AI account for parsing. No purchase
is required or offered anywhere in the app. All data is stored locally;
see the privacy policy. A demo requires no account: just type e.g.
"充电宝在卧室抽屉" (power bank in bedroom drawer) on the Record tab.
  • 被拒时先礼貌回复申诉(引用「app 免费、无解锁」),申诉不过再考虑 plan B:iOS 版隐藏自定义 endpoint 字段 / 把 AI 设置移到「高级」。
  • 别在截图和描述里出现「买 key / 充值」类字样 —— 描述里只说「支持接入你自己的 AI 服务」。
其余都是低风险:无收集(隐私问卷选 Data Not Collected 即可)、加密豁免已声明、无第三方 SDK、4+ 分级。

⑨ 中国大陆区 & ICP 备案

  • 2023-09 起新 app 上架中国大陆区必须提供 ICP 备案号,苹果强校验。
  • 建议:首发不选大陆区(港澳台 + 海外全开)。大陆用户装 app 常用外区账号,不影响你自己用(直接装本地构建即可)。
  • 以后想上大陆区:通过阿里云/腾讯云提交「App 备案」(个人主体 + 工具类,通过率高,约 1–3 周),拿到备案号后在 App Store Connect 填入,再把大陆加回销售范围。
